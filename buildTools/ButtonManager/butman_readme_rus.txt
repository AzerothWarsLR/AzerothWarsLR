[ ОПИСАНИЕ ]
Button Manager позволяет добавить к изображению одну из рамок в стиле Warcraft III. Кроме того, программа обладает следующими возможностями:
- Поддержка форматов .bmp, .jpg, .tga, .psd, .blp, .png;
- Массовое преобразование иконок;
- Массовый импорт иконок в архив;
- Возможность подгонки изображения под размер 64х64;
- Возможность применения нестандартной рамки.



[ РАБОТА С МАСТЕРОМ ИКОНОК ]
Мастер иконок позволяет применять рамки на все изображения в выбранной папке. Укажите исходную папку, папку назначения (куда будут копироваться готовые иконки), выберите требуемые типы рамок (для множественного выбора удерживайте Ctrl или Shift) и нажмите кнопку "Создать иконки".

Созданные иконки можно тут же импортировать в карту. Для этого необходимо нажать кнопку "Импортировать...", выбрать исходную карту и имя для вновь создаваемой карты.
Внимание: файлы для импорта берутся из папки, указанной в поле "Папка назначения".



[ ОКНО НАСТРОЕК ]
==== Вкладка "Общее":

- Автоматически применять рамку -
При открытии изображения накладывает выбранную рамку.

- ...и сохранять в файл -
В добавление к предыдущей опции, сохраняет полученную иконку в файл, используя формат по умолчанию.

- Автоматически дополнять Disabled-вариантом -
При сохранении с Button, Autocast или Passive рамкой программа будет создавать неактивную иконку-пару для полученной иконки.

- Автоматически подгонять под размеры 64х64 -
При открытии файла автоматически уменьшает/увеличивает изображение до нужных пропорций.

- Папка с пользовательскими рамками -
Папка, в которой лежат нестандартные рамки.

- Использовать файл манифеста -
При включенной опции для оформления вида программы будет использоваться тема ОС. Необходим перезапуск программы.

- Языковой модуль -
Список доступных языков в программе.

==== Вкладка "Изображение":

- Название рамки -
Список доступных в программе рамок.

- Префикс рамки -
Строка, которая будет добавляться перед именем файла для изображения с соответствующей рамкой.

- Путь импорта -
Используется при импорте, указывает, куда следует положить иконку с соответствующей рамкой.

- Затемнение -
Влияет на степень затемнения особых участков (в основном, границ) при совмещении рамки с изображением.

- Сжимать изображение в следующие границы -
Позволяет подгонять изображение в область, указанную координатами. Используется в основном для специфических рамок.

- Применять сглаживание при сжатии -
Если включена предыдущая опция, то будет использован метод гладкой прорисовки изображения в область.

==== Вкладка "Форматы":

- По умолчанию -
Формат, который будет применяться при сохранении иконок без участия пользователя (например, в мастере иконок).

- Название рамки -
Список доступных в программе рамок.

- Число текстур -
Число генерируемых мипмапов для изображения (http://ru.wikipedia.org/wiki/Mipmap)

- Сжатый формат -
Изображение будет сохранено в формате jpeg. Он наиболее применяем для изображений с большим количеством цветов. Размер файлов таких изображений практически не меняется, если поместить его MPQ (или любой другой архив). Поэтому формат и назван "сжатым". Изображение может быть сохранено с определенным качеством 1-100%. Spec рекомендует использовать 60-80%, что позволяет добиться хорошего качества при небольшом размере файла.

- Объединять заголовки -
Формат BLP позволяет иметь один jpeg заголовок для всех мипмапов в файле, что позволяет немного уменьшить размер этого файла. Эта опция генерирует такой заголовок. Также, она не будет иметь эффекта, если в поле "Число текстур" стоит "1".

- Прогрессивное кодирование -
Изображение будет сохранено в прогрессивном формате. Размер может колебаться (как в большую, так и в меньшую сторону) по сравнению со стандартным кодированием, но в архивах степень сжатия такого файла будет больше.

- Палитровый формат -
Изображение будет сохранено в качестве набора индексов с указанием на палитру. Палитра расположена сразу после заголовка BLP и имеет определенное число записей (256). В процессе обработки, если текущий цвет не будет найден в палитре, то он заменяется на приближенный цвет из палитры.

- Сжимать палитру -
Обычно размер палитры фиксирован и равен 1024 байтам. Эта опция позволяет уменьшить число записей в палитре до числа цветов изображения. Пример: нам нужно сохранить в BLP изображение с белым текстом на черном фоне. Когда выбрана эта опция, размер палитры будет равняться всего 8 байтам (2 цвета * 4 байта). Так же, как и опция "Объединение заголовков", это уменьшает размер файлов на выходе.

- Диффузия ошибок -
Подразумевается алгоритм Floyd-Steinberg (http://en.wikipedia.org/wiki/Floyd-Steinberg_dithering). Обычно после конвертирования многоцветного изображения в палитровый формат появляются артефакты и изображение в целом выглядит плохо. Эта опция позволяет сделать плавные переходы, поэтому оно будет выглядет лучше. Файлы, сохраненные с этой опцией, занимают больше места в архиве, чем файлы, сохраненные без нее.

- Автоматическое определение качества -
Самостоятельно выбирает параметры качества для BLP, опираясь на количество цветов текущего изображения.

- Для всех рамок -
Применяет текущие настройки качества BLP ко всем рамкам в программе.

==== Вкладка "Мастер иконок":

- Обработка файлов с расширениями -
Позволяет указывать, изображения каких форматов следует обрабатывать.

- Не обрабатывать файлы с префиксами -
Если в исходной папке лежат файлы с префиксами (btn, disbtn, etc.), то мастер иконок не будет обрабатывать их.

- Обрабатывать файлы в подпапках -
Позволяет обрабатывать все файлы (а не только те, что лежат в корне) в выбранной папке.

- Использовать путь импорта для сохранения иконок -
Автоматически создает для иконок папки, указанные в поле "Путь импорта" в настройках рамки.



[ ПОЛЬЗОВАТЕЛЬСКИЕ РАМКИ ]
При изготовлении нестандартных рамок следует учитывать то, что пиксели с одинаковым значением интенсивности трех каналов (например, red = 255, green = 255, blue = 255) будут считаться затемняющими пикселями. При наложении на изображение эти пиксели будут затемнять его, в зависимости от величины их интенсивности (например, 255 - без изменений, 128 - затемнение на 50%, 0 - черный пиксель).

Нестандартная рамка может быть с дополнительным (альфа) каналом. В этом случае при наложении рамки на изображение будет перенесен и её альфа-канал.



[ О ПРОГРАММЕ ]
Автор программы: Shadow Daemon (также известный как Spec).
Программа использует библиотеку SFMPQ.DLL от ShadowFlare

- ScorpioN - первоначальная иконка программы и материал для тестирования;
- Mastro - множество интересных идей и тестирование ранних версий;
- Krol, ADOLF - несколько советов по улучшению интерфейса программы;
- NETRAT - основной тестер;
- FellGuard, Hunter, MartyrOfSorrow, ScorpioT1000, Paladon, Enein - идеи и тестирование;
- автору файла для открытия PSD (ник мне неизвестен);
- Magos - описание формата BLP;
- Markus F. X. J. Oberhumer & Laszlo Molnar - UPX;
- а также всем, кто сделал небольшой, но полезный вклад в развитие программы и поддержал автора.

Если вы нашли какие-либо ошибки в программе, или у вас есть идеи по её улучшению, пишите в тему программы на форуме http://xgm.ru/forum/showthread.php?t=13692
